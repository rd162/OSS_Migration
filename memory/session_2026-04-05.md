## 5. Critical Bug Fixed: DB Isolation

**Problem:** Integration tests call `db.drop_all()` at session end. The dev server and integration tests were sharing the same DB (`ttrss_test` on port 5433). Running integration tests wiped the dev server's admin user, prefs, and all schema → 500 errors on every login.

**Fix:**
- Created separate `ttrss_dev` database on the same Docker Postgres instance (port 5433)
- Dev server now uses `ttrss_dev` (bootstrapped with Alembic + admin/admin)
- Integration tests continue using `ttrss_test`
- Added `just dev` and `just dev-setup` recipes to justfile

**Justfile commands:**
```
just dev          # start dev server on ttrss_dev, port 5001
just dev-setup    # bootstrap ttrss_dev schema + admin user (run once)
just test         # unit tests (no Docker)
just test-blueprints  # blueprint tests
just test-int     # integration tests (ttrss_test DB)
just test-fe      # frontend Playwright tests (requires :5001)
just test-fe-headed   # same with visible browser
```

## 7. Dev Server Startup

```bash
# One-time setup (creates ttrss_dev DB + admin user):
just dev-setup

# Start dev server:
just dev
# OR manually:
DATABASE_URL=postgresql://ttrss_test:ttrss_test@localhost:5433/ttrss_dev \
REDIS_URL=redis://localhost:6380/1 \
SECRET_KEY=dev-local-only FEED_CRYPT_KEY="" RATELIMIT_ENABLED=false \
uv run flask --app "ttrss:create_app({'TESTING':True,'WTF_CSRF_ENABLED':False})" \
  run --host 0.0.0.0 --port 5001
```

Credentials: `admin` / `admin`

For articles to appear after subscribing, run Celery:
```bash
DATABASE_URL=postgresql://ttrss_test:ttrss_test@localhost:5433/ttrss_dev \
REDIS_URL=redis://localhost:6380/1 \
uv run celery -A ttrss.celery_app worker --loglevel=info
```
