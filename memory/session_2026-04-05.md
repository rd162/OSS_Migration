### Phase 5c ‚Äî Strict Tree-Sitter Audit (2026-04-05, commit d401daf)

Previous exact_function_audit.py had three methodological flaws:
1. Global bare-name fallback: `Pref_Labels::edit` matched via `pref/filters.php::edit` (false positive)
2. PHP end-lines approximated from "next function start" ‚Äî not from tree-sitter `end_point`
3. Python Source comment regex missed: `func1 / func2`, `line# ‚Äî name`, `+ addendum`, `(name/name, lines X-Y)` formats

**New tool**: `tools/graph_analysis/strict_function_audit.py`
- Parses PHP DIRECTLY with tree-sitter ‚Üí exact start AND end line per function
- Enhanced Python comment parser handles all real-world Source formats found in codebase
- File-specific matching only ‚Äî no global bare-name fallback
- Boundary mismatch detection (48 informational warnings, not errors)

**Two genuine errors fixed**:
- `blueprints/prefs/labels.py`: added `Eliminated:` comment for `Pref_Labels::edit` (HTML dialog)
- `tasks/housekeeping.py`: fixed `cleanup_tags` Source from wrong `rssfuncs.php` to correct `functions2.php:2030-2069`

### Phase 5d ‚Äî Semantic Verification Plan v2 ‚Üí v3 (2026-04-05)

v1 plan was superficial ‚Äî only ~100 of 472 functions named across 15 workstreams.

v2 fixed INVENTORY (26 WS, 511 targets, every function listed) but METHODOLOGY still
shallow: D1-D18 taxonomy only, 15-row traps table, no cross-function or integration
verification, no complexity triage, no behavioral contracts.

**Concrete sampling proved v2 inadequate:** Compared 6 PHP‚ÜíPython function pairs
(`catchup_feed`, `sanitize`, `queryFeedHeadlines`, `update_rss_feed`, `make_init_params`,
`get_pref`). Found 60+ discrepancies in 15+ categories v2 didn't even define:
- Content priority inversion (summary vs content reversed)
- GUID construction divergence (missing owner_uid prefix + SHA1)
- Systemic type coercion gap (get_pref returns raw string vs typed)
- SQL join topology (3-table cross-join vs 1-table subquery)
- SQL computation locus (DB NOW() vs app datetime.now())
- Transactional granularity (per-article vs per-feed commit)
- Field truncation (245 chars in PHP, unlimited in Python)
- Hook argument count mismatch (5 args in PHP, 4 in Python)
- Feature elimination without flags (Sphinx, PubSub, favicon, image cache)
- DOM model divergence (DOMDocument vs lxml fragment)
- Session state elimination (13 missing init_params keys)
- Error recovery model (return error vs raise/retry)
- In-memory cache elimination (prefs, feed XML)

v3 rewritten from scratch ‚Äî kept v2 inventory, replaced methodology:
- **40-category discrepancy taxonomy** (D01-D40) organized by domain, proven against real code
- **40-row semantic traps table** covering types, strings, regex, dates, DB, HTTP, architecture
- **8 integration pipeline checks** (feed update, search, API, auth, counters, OPML, digest, plugin)
- **Complexity-tiered triage**: 52 Tier 1 (deep audit), ~150 Tier 2, ~270 Tier 3, 37 models
- **Cross-function contract verification** at pipeline boundaries
- **Model deep check** beyond columns: FKs, cascades, computed properties, validators
