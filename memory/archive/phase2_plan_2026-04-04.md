## Critical Architectural Decisions

### articles/sanitize.py (Batch 2)
- `sanitize(html_str, ...) -> str`: pipeline = (1) parse lxml, (2) HOOK_SANITIZE via pm.hook (collecting), (3) strip_harmful_tags(), (4) serialize
- **HOOK_SANITIZE fires BEFORE strip_harmful_tags** — matches PHP functions2.php lines 919–931
- `strip_harmful_tags(doc, allowed_elements, disallowed_attributes) -> doc`: separate top-level callable (R13)
- `format_inline_player()` is ABSENT from sanitize.py — belongs to article rendering path Phase 3 (R14)
- Source: functions2.php:sanitize (lines 831–965), strip_harmful_tags (lines 967+)

### feed_tasks.py integration (Batch 2) — 4 TODOs
1. HOOK_FETCH_FEED (rssfuncs.php line 270): collecting pipeline on raw bytes, BEFORE feedparser.parse()
2. HOOK_FEED_PARSED (rssfuncs.php line 394): fire-and-forget collecting, AFTER feedparser.parse()
3. HOOK_ARTICLE_FILTER (rssfuncs.php line 687): collecting pipeline per entry, inside entry loop
4. _sanitize_html stub → delegates to `articles.sanitize.sanitize()`; stub deleted

### http/client.py (Batch 2)
- Two sections: sync URL utilities (Flask-safe) + `async fetch_file_contents()` (Celery-only, ADR-0015)
- _fetch_feed_async in feed_tasks.py NOT moved here — has ETag-specific logic
- Source: functions.php:fetch_file_contents (lines 197–365), functions2.php (lines 1210–1310)

### prefs/ops.py (Batch 3)
- `initialize_user_prefs(uid, profile_id=None)`: dual SQL path
  - profile_id=None → `WHERE profile IS NULL` (global prefs)
  - profile_id=N → `WHERE profile = :id` (named profile)
- Iterates ttrss_prefs rows; inserts missing rows only (idempotent)
- Source: functions.php:initialize_user_prefs (lines 639–688 or 920–980)

### auth/authenticate.py (Batch 4)
- `authenticate_user(login, password, check_only=False) -> bool`
  - SINGLE_USER_MODE: `session["auth_module"] = False`, uid=1, no plugin call
  - Normal: `pm.hook.hook_auth_user(login, password)` (firstresult); on success: `session["auth_module"] = type(plugin).__name__.lower()`
  - After login: update last_login → set session fields → `initialize_user_prefs(uid)` → `load_user_plugins(uid)`
  - Source: functions.php:authenticate_user (lines 706–771)
- `logout_user()`: flask_login.logout_user() + session.clear() + session.modified=True (→ Redis key deleted)
  - Source: functions.php:logout_user (lines 807–812)
- `initialize_user(uid)`: inserts REAL feed rows (not virtual):
  - `(owner_uid=uid, title='Tiny Tiny RSS: New Releases', feed_url='http://tt-rss.org/releases.rss')`
  - `(owner_uid=uid, title='Tiny Tiny RSS: Forum', feed_url='http://tt-rss.org/forum/rss.php')`
  - Source: functions.php:initialize_user (lines 796–805)

## Rule 10a Checkpoints (after EACH batch)

Each batch has an explicit CRITIC checklist before commit. Key per-batch items:

**Batch 1:** loader.py KIND constants match PHP (KIND_ALL=1, KIND_SYSTEM=2, KIND_USER=3); every function has # Source comment
**Batch 2:** HOOK_SANITIZE line # < strip_harmful_tags call line # in sanitize.py; format_inline_player absent; all 4 TODOs closed (grep "TODO Phase 2" = 0); HOOK_FETCH_FEED before feedparser.parse() in feed_tasks.py

**Batch 4:** session["auth_module"] set on every auth path; initialize_user inserts real feeds with exact URLs; logout_user deletes Redis key
