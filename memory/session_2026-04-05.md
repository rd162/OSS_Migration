### Phase 5c — Strict Tree-Sitter Audit (2026-04-05, commit d401daf)

Previous exact_function_audit.py had three methodological flaws:
1. Global bare-name fallback: `Pref_Labels::edit` matched via `pref/filters.php::edit` (false positive)
2. PHP end-lines approximated from "next function start" — not from tree-sitter `end_point`
3. Python Source comment regex missed: `func1 / func2`, `line# — name`, `+ addendum`, `(name/name, lines X-Y)` formats

**New tool**: `tools/graph_analysis/strict_function_audit.py`
- Parses PHP DIRECTLY with tree-sitter → exact start AND end line per function
- Enhanced Python comment parser handles all real-world Source formats found in codebase
- File-specific matching only — no global bare-name fallback
- Boundary mismatch detection (48 informational warnings, not errors)

**Two genuine errors fixed**:
- `blueprints/prefs/labels.py`: added `Eliminated:` comment for `Pref_Labels::edit` (HTML dialog)
- `tasks/housekeeping.py`: fixed `cleanup_tags` Source from wrong `rssfuncs.php` to correct `functions2.php:2030-2069`

**Final results**: 288/288 in-scope = 100%, 0 missing, 0 file-level-only. 602 tests pass.

- Hook argument count mismatch (5 args in PHP, 4 in Python)
- Feature elimination without flags (Sphinx, PubSub, favicon, image cache)
- DOM model divergence (DOMDocument vs lxml fragment)
- Session state elimination (13 missing init_params keys)
